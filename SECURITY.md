# Security Policy

## Supported versions

| Version | Supported |
|---|---|
| 1.x | ✅ |

## Reporting a vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

Email details to the repository maintainers or open a private security advisory on GitHub.

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We aim to acknowledge reports within 72 hours.

## Scope

- Credential leakage patterns in SDK usage
- HMAC implementation flaws
- Cache poisoning or deserialization issues
- Supply chain concerns in published packages

Out of scope: vulnerabilities in the onOffice API server itself — report those to [apisupport@onoffice.de](mailto:apisupport@onoffice.de).
